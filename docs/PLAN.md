# Storlit - 릴레이 소설 서비스 기획 및 MVP 계획

> 인터넷 커뮤니티의 1회성 글과 댓글에서 발견되는 집단 창의성을 "릴레이 소설"이라는 형식으로 구조화하여, 매일 한 편의 단편이 완성되는 참여형 창작 플랫폼

## 서비스 개요

1. 정해진 시간이 되면, 시작점이 되는 몇 문장의 짧은 글이 제시된다.
2. 사람들은 1시간 동안 다음에 이어질 내용을 정해진 분량 안에서 작성한다.
3. 작성된 글들 중 가장 많은 좋아요(반응)를 받은 글이 다음 내용이 된다.
4. 이 과정을 1시간마다 반복하여 하루가 지나면 한 편의 단편이 완성된다.
5. 매일 반복되어 매일 한 편의 단편이 만들어지는 공간이 된다.

## 1. 서비스 장점 분석

### 집단 창작의 재미
- 혼자 쓰는 것보다 예측 불가능한 전개가 만들어지는 재미
- 커뮤니티 댓글 문화의 창의성을 생산적 결과물로 전환
- "내가 쓴 문장이 채택되었다"는 성취감과 소속감

### 낮은 참여 장벽
- 1~2문장만 쓰면 되므로 글쓰기 부담이 극히 낮음
- 투표만으로도 참여 가능 → 소비자도 참여자가 됨
- 별도 앱 설치 없이 모바일 웹으로 즉시 접근

### 일일 콘텐츠 자동 생성
- 매일 1편씩 자동으로 콘텐츠가 축적됨
- 완성된 단편은 아카이브로 지속 소비 가능
- 운영 측에서 별도 콘텐츠 제작 비용 없이 콘텐츠 확보

### 시간 제한의 긴장감
- 1시간 라운드가 게임적 긴장감 부여
- 정해진 시간에 모이는 습관 형성 → 리텐션 유도
- 카운트다운이 만드는 긴박함과 몰입감

### 네트워크 효과
- 참여자가 많을수록 선택지 품질 상승
- 좋은 작품이 나올수록 공유/바이럴 가능성 증가
- 작가 커뮤니티 형성 → 자체적인 팬덤 생태계

## 2. 보완해야 할 부분

### 초기 참여자 부족 (Cold Start)
- **문제**: 참여자가 적으면 선택지 품질이 낮아지고 재미가 떨어짐
- **제안**: 초기에는 라운드 시간을 2~3시간으로 늘리거나, AI가 보조 선택지를 생성하여 최소 품질 보장

### 투표 조작 / 어뷰징
- **문제**: 다중 계정으로 자기 글에 투표하는 행위
- **제안**: 소셜 로그인 필수 + 일정 참여 이력이 있는 유저만 투표 가능 + IP 기반 중복 감지

### 문맥 일관성
- **문제**: 릴레이 특성상 앞뒤 문맥이 맞지 않거나 trolling 글이 선택될 수 있음
- **제안**: 신고 기능 + 일정 비율 이상 신고 시 자동 필터링 + 커뮤니티 가이드라인 마련

### 시간대별 참여 불균형
- **문제**: 새벽 라운드에는 참여자가 거의 없을 수 있음
- **제안**: 활성 시간대(오전 9시~자정)에만 라운드 운영, 나머지 시간은 휴식 또는 투표 연장 시간으로 활용

### 25문장 고정의 경직성
- **문제**: 이야기에 따라 25문장이 너무 길거나 짧을 수 있음
- **제안**: 15~25문장 유동적 운영, 또는 "엔딩 투표" 기능으로 참여자가 마무리 시점을 결정

## 3. 추가 기능 아이디어

### 장르/테마 투표
- 매일 새로운 이야기 시작 전, 오늘의 장르(로맨스, 공포, 코미디, SF 등)를 투표로 결정
- 특정 요일에 특정 장르를 고정하는 것도 가능 (예: 금요일은 공포의 날)

### 작가 프로필 & 랭킹
- 채택 횟수 기반 작가 레벨/뱃지 시스템
- 주간/월간 베스트 작가 랭킹
- 작가별 채택된 문장 모아보기

### 분기형 스토리 (평행 세계)
- 1위 선택지 외에 2위 선택지로 갈라지는 "평행 세계" 버전 제공
- 같은 시작점에서 다른 결말이 만들어지는 재미
- 독자가 두 버전을 비교하며 읽는 새로운 경험

### AI 일러스트레이션
- 완성된 단편에 AI 생성 일러스트를 자동 첨부
- 각 라운드별 장면 이미지 생성으로 시각적 몰입감 제공

### 소설 공유 & 굿즈화
- 완성된 단편을 카드뉴스/이미지 형태로 SNS 공유
- 인기작을 e-book/포스터 등으로 제작
- 참여 작가에게 수익 분배

### 실시간 리액션
- 글 작성 중 실시간으로 다른 사람들의 리액션 확인
- 라운드 종료 직전 카운트다운 + 실시간 투표 현황 공개

### 시즌제 운영
- 주간/월간 시즌으로 묶어서 시즌 베스트 작품 선정
- 시즌별 특별 테마 이벤트 운영

## 4. MVP 구현 계획

### 4.1 기술 스택

| 영역 | 기술 | 선택 이유 |
|------|------|-----------|
| Frontend | Next.js 14+ (App Router) + TypeScript | SSR/PWA 지원, 모바일 웹 최적화 |
| 스타일링 | Tailwind CSS | 빠른 모바일 UI 개발 |
| Backend | Next.js API Routes | 프론트와 통합, 서버리스 |
| DB & Auth | Supabase (PostgreSQL) | BaaS로 빠른 개발, 소셜 로그인 내장, Realtime 지원 |
| 상태관리 | Zustand | 가볍고 간결한 상태관리 |
| 배포 | Vercel | Next.js 최적 배포, Cron Jobs 지원 |

### 4.2 데이터 모델

```
User
├── id (uuid, PK)
├── nickname (varchar)
├── email (varchar)
├── provider (varchar) — kakao, google 등
├── avatar_url (varchar, nullable)
└── created_at (timestamp)

Story
├── id (uuid, PK)
├── title (varchar)
├── genre (varchar, nullable)
├── status (enum: in_progress, completed)
├── started_at (timestamp)
└── completed_at (timestamp, nullable)

Round
├── id (uuid, PK)
├── story_id (uuid, FK → Story)
├── round_number (int)
├── started_at (timestamp)
├── ends_at (timestamp)
├── status (enum: active, voting, completed)
└── winning_submission_id (uuid, FK → Submission, nullable)

Submission
├── id (uuid, PK)
├── round_id (uuid, FK → Round)
├── user_id (uuid, FK → User)
├── content (text)
├── vote_count (int, default 0)
└── created_at (timestamp)

Vote
├── id (uuid, PK)
├── submission_id (uuid, FK → Submission)
├── user_id (uuid, FK → User)
└── created_at (timestamp)
└── UNIQUE(submission_id, user_id)
```

### 4.3 핵심 API

| Method | Endpoint | 설명 |
|--------|----------|------|
| POST | /api/auth/* | 소셜 로그인 (Supabase Auth) |
| GET | /api/stories/current | 현재 진행 중인 스토리 |
| GET | /api/stories/[id] | 완성된 스토리 상세 조회 |
| GET | /api/stories | 완성된 스토리 목록 (아카이브) |
| GET | /api/rounds/current | 현재 라운드 정보 + 제출글 목록 |
| POST | /api/submissions | 글 제출 (라운드당 1회) |
| POST | /api/votes | 투표 (제출당 1회) |
| DELETE | /api/votes/[id] | 투표 취소 |
| CRON | /api/cron/advance-round | 라운드 자동 진행 (1시간마다) |

### 4.4 화면 구성

#### 1) 메인 화면 (현재 라운드)
- 지금까지 완성된 스토리 내용 표시
- 현재 라운드에 제출된 글 목록 + 투표 버튼
- 남은 시간 카운트다운 타이머
- 글쓰기 버튼 (라운드 중일 때만 활성)

#### 2) 글쓰기 화면
- 이전까지의 스토리 내용 표시
- 텍스트 입력 영역 (글자 수 제한 표시)
- 제출 버튼

#### 3) 아카이브 화면
- 완성된 단편 목록 (날짜순)
- 클릭 시 전체 단편 읽기
- 공유 버튼

#### 4) 프로필 화면
- 내가 쓴 글 / 채택된 글 목록
- 채택 횟수 통계
- 로그인 / 로그아웃
